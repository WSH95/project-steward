Recommend, explain, or switch the Project Steward task backend. If the
backend-broker skill is installed, follow it; otherwise use this
condensed protocol.

1. Run `project-steward backend recommend` if the CLI is installed.
2. Explain the top options in plain English for a user who may not know
   Markdown tasks, Backlog.md, beads, CCPM, Taskmaster, Spec Kit, or
   GitHub Issues.
3. Get approval before switching.
4. Run `project-steward backend adopt <name>` only after approval, then
   help move open tasks so one system owns fine-grained work and PLAN.md
   becomes milestones plus a pointer.
5. Never install anything without explicit approval.
